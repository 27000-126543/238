# 大型电网调度与设备运维桌面系统

## 系统简介

本系统是一个功能完善的电网调度与设备运维管理平台，基于 Electron + Vue 3 + TypeScript + Element Plus 技术栈开发，支持跨平台运行。

## 主要功能模块

### 1. 运行总览 (Dashboard)
- 系统核心指标实时展示（发电量、负荷、频率、设备可用率）
- 负荷与发电趋势图
- 电源结构饼图
- 最新告警列表
- 设备状态概览

### 2. 设备管理 (Equipment)
- **发电厂管理**: 支持火电、水电、核电、风电、光伏电站信息录入
  - 名称、容量、额定参数（电压、电流）
  - 最大/最小出力、爬坡速率
  - 当前状态、运行时长、启停次数
- **变电站管理**: 电压等级、容量、变压器数量、母线电压
- **输电线路管理**: 起点/终点变电站、电压等级、长度、潮流、负载率
- 支持新增、编辑、删除操作

### 3. 发电调度 (Dispatch)
- **智能调度方案生成**:
  - 基于历史负荷、天气预报数据
  - 考虑检修计划约束
  - 自动生成未来24小时出力计划
- **安全约束校验**:
  - N-1准则校验（任意单线路停运仍满足供电）
  - 机组爬坡速率校验
  - 功率平衡校验
- **审批流程**:
  - 方案提交总调度长审批
  - 支持通过/驳回操作
- **指令推送**:
  - 审批通过后推送指令至厂站终端
  - 值班员确认或申请调整
  - 调整申请需总调审批

### 4. 实时监控 (Monitor)
- 系统频率、平均电压、总出力实时监测
- 频率与负荷趋势图（60个历史数据点）
- 分级告警机制：
  - 告警级别：警告、危险、紧急
  - 告警类型：频率异常、电压超限、潮流过载、设备故障
  - 告警确认与处理流程
- 线路潮流监控（负载率可视化）
- 电厂出力监控
- **应急预案**:
  - 切负荷预案（频率<49.5Hz触发）
  - 甩负荷预案（线路过载>95%）
  - 解列预案（系统失步）

### 5. 电网拓扑 (Topology)
- **可视化拓扑图**:
  - 节点类型区分（发电厂/变电站）
  - 状态颜色标识
  - 支持拖拽、缩放
  - 点击节点查看详情
- **两种视图模式**:
  - 电压分布视图：节点颜色反映电压水平
  - 负载热力视图：线路颜色反映负载率
- 线路负载列表详情

### 6. 维保管理 (Maintenance)
- **工单自动生成**:
  - 基于设备运行时长（>8000小时）
  - 基于设备状态告警（线路负载过高）
- 工单类型：常规、预防性、故障修复
- 优先级：低、中、高、紧急
- **班组分配**:
  - 选择维修班组
  - 自动扣减备件库存
- **备件库存管理**:
  - 库存数量、安全库存预警
  - 库位管理
- 维修班组管理
- 检修计划管理

### 7. 人员排班 (Scheduling)
- **自动排班**:
  - 基于人员技能匹配
  - 考虑工时上限约束
  - 按周生成排班表
- 班次：早班(08:00-16:00)、中班(16:00-24:00)、夜班(00:00-08:00)
- **调班功能**:
  - 员工提交调班申请
  - 管理员审批
  - 自动更新排班表
- 人员信息管理（技能、工时、状态）

### 8. 统计报表 (Statistics)
- **核心指标统计**:
  - 总发电量、总供电量
  - 线损率
  - 设备可用率
  - 故障/事故次数
  - 峰谷负荷
- 支持按区域、时间段查询
- **图表可视化**:
  - 发电量趋势柱状图
  - 区域发电量对比饼图
  - 线损率趋势图
  - 设备类型统计
- **PDF导出**:
  - 一键导出月度运行报告
  - 包含汇总数据和详细表格
  - 自动生成页码和时间戳

## 技术栈

- **前端框架**: Vue 3 (Composition API)
- **类型系统**: TypeScript
- **UI组件库**: Element Plus
- **路由管理**: Vue Router 4
- **状态管理**: Pinia
- **图表可视化**: ECharts 5
- **日期处理**: Day.js
- **PDF导出**: jsPDF + jspdf-autotable
- **桌面框架**: Electron
- **数据存储**: LocalStorage (可扩展为 SQLite)
- **构建工具**: Vite

## 快速开始

### 环境要求
- Node.js >= 18.0.0
- npm >= 9.0.0

### 安装依赖
```bash
npm install
```

### 开发模式运行
```bash
# Web 模式
npm run dev

# Electron 桌面模式
npm run electron:dev
```

### 构建打包
```bash
# Web 构建
npm run build

# Electron 桌面应用构建
npm run electron:build
```

## 核心算法说明

### 调度算法
1. **经济调度**: 按发电成本排序（核电<水电<光伏<风电<火电）
2. **负荷分配**: 按小时负荷预测值分配各机组出力
3. **爬坡约束**: 相邻小时出力变化不超过机组爬坡速率
4. **N-1校验**: 模拟单线路停运，验证剩余网络容量裕度

### 排班算法
1. 筛选在岗人员
2. 按技能匹配度排序
3. 考虑周工时上限
4. 循环分配各时段班次

## 数据持久化

3. 实现数据备份与恢复
